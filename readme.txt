2.概要

	・DirextXのエンジンを使用してアルティメットチキンホースとStick Fight: The Gameを参考にして作った3Dアクションゲームです。
 	・最大4人で戦い、試合をする度に設置するオブジェクトを選んで、それをステージに設置をしてまた試合を繰り返すゲームです。
	・勝利やキルでもらえるポイントが一定まで一番早く溜まったプレイヤーが優勝です。

3.開発環境

　　　・ハードウェア      : WindowsPC(i7-12700H、メモリ32GB)
　　　・開発用ＯＳ        : Windows11
　　　・統合開発環境	： Microsoft Visual Studio 2022
　　　・開発言語          ： C / C++ / HLSL
　　　・ライブラリ        : directX11 / FBXSDK

4.操作説明

　　　・入力デバイス : マウス、キーボード

試合中での操作

　　　・WASD    : 移動
　　　・Fキー   ：攻撃
　　　・スペースキー：結果等のゲージを飛ばしたり、要所要所で次に進むために使います


クリエイトモードでの操作
　　　・マウスカーソル　：　設置したいオブジェクトを選択、設置したい場所に合わせて設置
　　　・左クリック　　　：　オブジェクトを選択、設置
　　　・矢印キー　　　　：　オブジェクトの向きを変える

5.作品に対するアピールポイント
　　　
　　　・ゲームAI(メタAI,キャラAI,ナビAI)を使用してCPUが自立して移動、攻撃、オブジェクトの設置を行うようにしました。
　　　・Astarアルゴリズムを使い、CPUが壁等を認識してなるべく避けるようにプログラムしました。
　　　・CPUとプレイヤーのどちらにも共通するstateパターンを作る事で、可読性を上げました。
　　　・enumの数分の大きさの配列を作り、swich分を使わなくてもstateの切り替えができるようにしました。
　　　・クリエイトモードで、プレイヤーが自由にオブジェクトの設置と移動ができるようにしました。
　　　・途中経過等のUIをパッとわかるように凝りました。
　　　・アニメーションがなるべく自然な動きになるようにmayaで丁寧に作りました。
　　　・定数宣言を無名名前空間で行い、値が楽にいじれるようにしました。
　　　・クリエイトモードでビリの人から優先してオブジェクトを選べるようにしました。


6.反省点、改善点
　　　・UIの実装がかなりごり押しになってしまった。
	→もっとUI全体で使えるような基底クラスを作って、画像をいじりやすくするべきだった

　　　・プランナーが値をいじる事が難しくなってしまった。
	→ImGuiライブラリとjsonファイルを使い、値をいじれる様にするべきだった。
